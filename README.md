# まちあるき学習アプリ（MVP）

街を歩きながら学べる、フィールドワーク用のWebアプリです。
**先生がコースを作り → 生徒が現地でスマホを使って学習**します。

- 地図上のスポット巡り（Leaflet + OpenStreetMap、APIキー不要）
- GPSでスポットに近づくと「到着」判定 → 課題・クイズが開く
- 現地で写真・メモを記録（端末内に保存）
- ルート表示と進捗バー、ふりかえり画面
- **サーバー・データベース不要**。静的ファイルだけで動きます。

## ファイル構成
```
machiaruki-app/
├─ index.html      生徒用アプリ
├─ editor.html     先生用コース作成エディタ
├─ student.js      生徒用ロジック
├─ editor.js       先生用ロジック
├─ common.js       共通（地図・保存・共有リンク）
├─ app.css         デザイン
└─ courses/
   └─ sample-naha.json  サンプルコース
```

## 動かし方（ローカル）

GPS・カメラを使うため、`file://` で直接開くのではなく
**ローカルサーバー経由**（または `localhost`）で開く必要があります。

ターミナルでこのフォルダに入って:

```bash
cd machiaruki-app
python3 -m http.server 8000
```

ブラウザで開く:
- 生徒用（サンプル）: http://localhost:8000/index.html?course=sample-naha
- 先生用エディタ: http://localhost:8000/editor.html

※ スマホで試すときは、同じWi‑Fi内でPCのIPアドレス（例: `http://192.168.0.5:8000/...`）にアクセス。
　ただしGPS/カメラは **httpsまたはlocalhostでないと動かない** ブラウザがあります（下の「公開」参照）。

## 使い方

### 先生（コース作成）
1. `editor.html` を開く
2. コース名を入力
3. 地図をクリックしてスポットを追加 → 説明・課題・クイズを入力して「保存」
4. 「🔗 生徒用リンクをコピー」 → LINEやメールで生徒に配布
   （リンクの中にコース情報が全部入っているので、サーバー不要）

### 生徒（現地で）
1. 配布されたリンクを開く
2. 「📍 現在地へ」で位置情報をON
3. スポットに近づくと自動で「到着」→ 課題・クイズに挑戦
4. 「ふりかえり」で記録を確認・書き出し

> 室内で試すときは「🔓 練習モード」をONにすると、GPSなしで開錠できます。

## インターネット公開（無料）

GPS/カメラはhttpsが必要なので、公開して使うのがおすすめです。
このフォルダごとアップロードするだけ:

- **Netlify Drop**: https://app.netlify.com/drop にフォルダをドラッグ
- **GitHub Pages**: リポジトリにpush → Settings > Pages を有効化
- **Cloudflare Pages / Vercel** などでも可

公開後のURL例:
- 先生用: `https://あなたのサイト/editor.html`
- 生徒用リンクは先生用画面の「生徒用リンクをコピー」で生成

## データの保存場所
- 生徒の記録（到着・メモ・クイズ）= ブラウザの localStorage
- 写真 = ブラウザの IndexedDB（自動で圧縮して保存）
- いずれも **その端末・そのブラウザの中だけ** に保存されます（外部送信なし）
